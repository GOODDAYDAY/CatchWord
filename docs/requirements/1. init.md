# REQ-001: CatchWord 初始化 — 双击翻译器 + 单词本

**状态：** 已完成

## 概述

构建一个 Tauri v2 桌面应用，实现系统全局双击取词翻译功能，类似有道词典。用户在任意应用（浏览器、编辑器、PDF、终端等）中双击选中英文单词，即可弹出翻译浮窗，展示翻译结果并播放发音。查询过的单词自动存入本地单词本。

## 产品形态

- **类型：** Tauri v2 桌面应用（Rust + WebView2）
- **取词范围：** 系统全局（任意应用中双击或拖选英文单词）
- **展示形式：** 鼠标旁弹出翻译浮窗（直角边框、无系统装饰、置顶显示）
- **语言方向：** 英 → 中
- **平台：** Windows 10/11
- **功能开关：** 所有功能均可通过系统托盘右键菜单独立开关

## 核心功能

### F1: 系统全局选词取词

- 使用 `rdev` crate 监听系统全局鼠标事件（ButtonPress / ButtonRelease）
- 检测双击（500ms 内两次点击）和拖选（移动超过 5px 阈值）
- 触发后使用两级策略捕获选中文本：
  - **策略 A — Windows UI Automation（UIA）：** 无副作用，直接读取控件选中文本
    - 策略 1：焦点元素的 TextPattern（适用于终端、编辑器）
    - 策略 2：向上遍历祖先元素找 TextPattern（适用于浏览器）
    - 策略 3：向下遍历子元素找 TextPattern（特殊控件）
  - **策略 B — 截图 + Windows OCR 兜底：** 适用于 UIA 不支持的应用（PDF 等）
    - GDI 截取鼠标附近 300×60 像素区域
    - 使用 Windows.Media.Ocr（系统自带英文 OCR，离线可用）识别文字
    - 返回距鼠标中心最近的单词
- 判断捕获文本是否为英文单词（2-45 字符，仅含字母/连字符/撇号）
- **安全约束：** 绝对禁止键盘模拟（enigo）和剪贴板操作（arboard），防止向非目标窗口发送不可控快捷键
- **托盘开关：** 取词：已开启 / 已关闭

### F2: 翻译查询

- 调用 Google Translate 免费端点（`translate.googleapis.com`），无需 API Key
- 请求参数：`client=gtx&sl=en&tl=zh-CN&dt=t&dt=bd&dt=rm`
- 解析返回 JSON 获取：翻译文本、音标、词性释义
- 5 秒超时，失败时静默放弃

### F3: 翻译浮窗

- 在鼠标位置偏移 15px 处弹出浮窗
- 使用物理像素坐标定位（兼容高 DPI 屏幕）
- 浮窗内容：
  - 单词原文
  - 音标 + 发音按钮
  - 中文释义
  - 词性释义列表（最多 6 条）
  - 例句（最多 2 条）
- 窗口大小由前端 JS 根据内容动态调整
- 点击浮窗外部区域自动隐藏
- 浮窗配置：`alwaysOnTop`、无装饰、无阴影、`skipTaskbar`、直角边框
- **绝不调用 `set_focus()`**，避免抢夺用户应用焦点

### F4: 发音播放

- 翻译结果展示后自动播放单词发音
- 提供手动播放按钮（可重复点击）
- 使用有道词典 TTS：`https://dict.youdao.com/dictvoice?audio={word}&type=2`
- **托盘开关：** 自动发音：已开启 / 已关闭（关闭后仍可手动点击播放）

### F5: 单词本

- 每次查询自动将单词存入本地单词本
- **存储路径：** `%APPDATA%/com.catchword.app/wordbook.json`
- **存储格式：** JSON 数组
- **单词条目字段：**
  - `word`: 单词原文（小写）
  - `translation`: 中文释义
  - `phonetic`: 音标
  - `examples`: 例句列表
  - `query_time`: 首次查询时间（RFC 3339）
  - `last_query_time`: 最近查询时间（RFC 3339）
  - `source_context`: 来源上下文
  - `query_count`: 查询次数（重复查询时递增）
  - `mastered`: 是否已掌握（默认 false）
- 重复查询同一单词时更新 `query_count` 和 `last_query_time`，不重复添加
- **托盘开关：** 自动记单词：已开启 / 已关闭

## 系统托盘

右键托盘图标显示菜单，所有功能均可独立开关：

| 菜单项            | 说明               |
|----------------|------------------|
| 取词：已开启/已关闭     | 总开关，关闭后不监听选词事件   |
| 自动发音：已开启/已关闭   | 关闭后翻译不自动播放，仍可手动点 |
| OCR 兜底：已开启/已关闭 | 关闭后 UIA 失败时不截图识别 |
| 自动记单词：已开启/已关闭  | 关闭后查词不写入单词本      |
| 退出             | 关闭应用             |

## 验收标准

1. 应用启动后，在浏览器/编辑器/终端中双击英文单词，能弹出翻译浮窗（UIA 策略）
2. 在 PDF 阅读器中双击单词，能通过 OCR 兜底捕获并翻译
3. 浮窗显示单词、音标、中文释义、词性、例句
4. 浮窗为直角边框，无系统装饰，置顶但不抢焦点
5. 翻译完成后自动播放发音，也可手动点击播放
6. 每次查询的单词自动保存到本地 JSON 单词本
7. 重复查询同一单词时，查询次数递增而非重复添加
8. 点击浮窗外部可关闭浮窗
9. 点击浮窗内的播放按钮不会触发新的取词流程
10. 任何情况下不会向用户应用发送不可控的快捷键
11. 系统托盘可独立开关：取词、自动发音、OCR 兜底、自动记单词
12. 开发模式有自测浮窗，生产模式无

## 技术栈

| 层        | 技术                                         |
|----------|--------------------------------------------|
| 框架       | Tauri v2（Rust + WebView2）                  |
| 前端       | HTML / CSS / JS + Vite                     |
| 鼠标监听     | `rdev` crate                               |
| 取词 — 主策略 | Windows UI Automation API（`windows` crate） |
| 取词 — 兜底  | GDI 截图 + Windows.Media.Ocr                 |
| 翻译 API   | Google Translate 免费端点（无需 API Key）          |
| 发音       | 有道词典 TTS                                   |
| 单词本      | 本地 JSON 文件（`serde_json`）                   |

## 技术约束

- **禁止使用** `enigo`（键盘模拟）、`arboard`（剪贴板操作）及任何形式的 Ctrl+C 方案
- UIA 客户端必须使用 `COINIT_MULTITHREADED`（MTA）
- 浮窗绝不调用 `set_focus()`
- 处理 PossibleSelection 事件前必须先检查点击是否在浮窗内
- MVP 阶段不需要用户登录、云同步等功能
