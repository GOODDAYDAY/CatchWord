# REQ-001: CatchWord 初始化 — 双击翻译器 + 单词本

## 概述

构建一个 Electron 桌面应用，实现系统全局双击取词翻译功能，类似有道词典。用户在任意应用（浏览器、PDF、Word 等）中双击选中单词，即可弹出翻译浮窗，展示翻译结果并播放发音。查询过的单词自动存入本地单词本。

## 产品形态

- **类型：** Electron 桌面应用
- **取词范围：** 系统全局（任意应用中双击选中单词）
- **展示形式：** 浮窗弹出（在鼠标位置附近弹出翻译结果）
- **语言方向：** 英 → 中

## 核心功能

### F1: 系统全局选词取词

- 监听系统全局的鼠标事件（mouseup）
- 检测鼠标松开后是否有新选中的文本（通过模拟 Ctrl+C + 读取剪贴板获取）
- 无论用户是双击选词还是拖拽选中都能触发
- 选中文本前保存剪贴板内容，取词后恢复
- 如果选中的文本是英文单词，触发翻译流程
- Windows 平台优先支持

### F2: 翻译查询

- **MVP 阶段：** 使用 google-translate-api 等免费库（无需 API Key）
- **后续：** 可切换到 Google Translate 官方 API
- 查询英文单词的中文释义
- 获取音标信息
- 翻译接口需设计为可替换的抽象层，方便后续切换

### F3: 翻译浮窗

- 在鼠标双击位置附近弹出浮窗
- 浮窗内容：
  - 单词原文
  - 音标 + 发音按钮
  - 中文释义
  - 例句（如 API 提供）
- 点击浮窗外部区域关闭浮窗
- 浮窗置顶于其他应用之上

### F4: 发音播放

- 翻译结果展示后自动播放单词发音
- 提供手动播放按钮
- 使用 Google Translate TTS URL 直接获取音频流播放

### F5: 单词本

- 每次查询自动将单词存入本地单词本
- **存储格式：** 本地 JSON 文件
- **单词条目字段：**
  - `word`: 单词原文
  - `translation`: 中文释义
  - `phonetic`: 音标
  - `examples`: 例句列表
  - `queryTime`: 首次查询时间
  - `lastQueryTime`: 最近查询时间
  - `sourceContext`: 来源句子（双击时所在的上下文，如果能获取的话）
  - `queryCount`: 查询次数（重复查询时递增）
  - `mastered`: 是否已掌握（布尔值，默认 false）

## 验收标准

1. 应用启动后，在任意应用中双击英文单词，能弹出翻译浮窗
2. 浮窗显示单词、音标、中文释义
3. 翻译完成后自动播放发音，也可手动点击播放
4. 每次查询的单词自动保存到本地 JSON 单词本
5. 重复查询同一单词时，查询次数递增而非重复添加
6. 点击浮窗外部可关闭浮窗
7. 应用可正常启动和退出

## 技术约束

- Tauri v2 + Rust（后端）+ HTML/CSS/JS（前端）
- 系统全局取词使用 `rdev` crate 监听鼠标事件 + `enigo` 模拟 Ctrl+C + `arboard` 读取剪贴板
- 翻译 API 直接调用 Google Translate 非官方端点（`translate.googleapis.com`），无需 API Key
- MVP 阶段不需要用户登录、云同步等功能
- 单词本数据存储在用户本地文件系统（JSON）

## 原始需求

> 我想做的是一个翻译器，其实跟有道云词典的那种双击可以获取单词的功能差不多。这个翻译器的核心功能是能够在用户双击一个单词时，自动获取该单词的翻译结果并显示出来。我需要在翻译展示之后还要播放音频。然后查询的单词，要放入单词本中，当前单词本就是本地json啥的就可以了。
