# REQ-004: 多翻译源支持

## 概述

当前仅使用 Google Translate 免费端点，在国内可能无法访问。支持多个免费翻译源，提供「海外/大陆」两种模式预设，用户也可手动切换具体翻译源。翻译失败时自动回退到下一个源。

## 状态

Done

## 翻译源

全部使用免费端点，无需 API Key：

| 源                | 端点                                           | 国内可用 | 数据丰富度       |
|------------------|----------------------------------------------|------|-------------|
| Google Translate | `translate.googleapis.com`                   | 需梯子  | 高（音标、词性、释义） |
| Bing Translator  | `api-edge.cognitive.microsofttranslator.com` | 可用   | 中（释义）       |
| MyMemory         | `api.mymemory.translated.net`                | 可用   | 低（仅翻译）      |

## 模式预设

| 模式   | 默认源    | 回退顺序                     |
|------|--------|--------------------------|
| 海外模式 | Google | Google → Bing → MyMemory |
| 大陆模式 | Bing   | Bing → MyMemory          |

选模式会自动切换默认翻译源和回退链。切换模式后仍可手动指定具体翻译源。

## TTS 发音

统一使用有道词典 TTS（`dict.youdao.com/dictvoice`），国内外均可用，不受翻译源影响。

## 托盘菜单

```
翻译模式
  ✓ 海外模式
    大陆模式
翻译源
  ✓ Google Translate
    Bing Translator
    MyMemory
──────────
取词：已开启
自动发音：已开启
...
```

- 「翻译模式」子菜单：选模式，自动设置默认源
- 「翻译源」子菜单：手动指定具体翻译源，覆盖模式默认值
- 当前选中项打勾
- 选择持久保存，重启后保持

## 回退机制

翻译请求失败时（超时 5s / HTTP 错误），按当前模式的回退顺序自动尝试下一个源，直到成功或全部失败。全部失败时在浮窗显示错误提示。

## 数据适配

不同翻译源返回格式不同，统一适配到现有 `TranslationResult`：

- `word` — 所有源都有
- `phonetic` — 仅 Google 有，其他源留空
- `translation` — 所有源都有
- `definitions` — Google 有词性+释义，Bing 有释义，MyMemory 无
- `examples` — 目前都留空
- `audio_url` — 统一使用有道 TTS

- [x] 托盘菜单有「翻译源」子菜单，可手动选择具体源
- [x] 切换模式自动更新默认翻译源
- [x] Google、Bing、MyMemory 三个源均可正常翻译
- [x] 翻译失败自动回退到下一个源
- [x] 选择的模式和翻译源重启后保持
- [x] 所有源的结果统一显示在浮窗中，不会因字段缺失而报错
