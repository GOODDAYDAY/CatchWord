# REQ-003: GitHub Actions CI/CD

## 概述

利用 GitHub Actions 自动化构建和发布 Windows 安装包。在 GitHub 创建 Release 时，自动构建 NSIS 安装包和 Portable 绿色版，上传到该
Release。

## 状态

Done (2026-02-28)

## 触发条件

- 推送 `v*` 格式的 tag 时自动触发（`on: push: tags: ["v*"]`）
- CI 自动创建 GitHub Release 并上传构建产物

## 构建环境

- Runner: `windows-latest`
- Node.js 22 + npm
- Rust stable
- 使用 `swatinem/rust-cache@v2` 缓存 Rust 编译产物

## 构建步骤

1. Checkout 代码
2. 安装 Node.js 22，缓存 npm 依赖
3. 安装 Rust stable 工具链
4. `npm ci` 安装前端依赖
5. `tauri-apps/tauri-action@v0` 构建 NSIS 安装包，自动创建 Release 并上传
6. 将 `CatchWord.exe` 打包为 Portable ZIP 并追加上传到 Release

## 产物

| 文件                                  | 说明                          |
|-------------------------------------|-----------------------------|
| `CatchWord_{version}_x64-setup.exe` | NSIS 安装包（tauri-action 自动上传） |
| `CatchWord-portable-{tag}.zip`      | 绿色便携版                       |

## 配置变更

- `tauri.conf.json` 的 `bundle.targets` 从 `"all"` 改为 `["nsis"]`，只生成 NSIS 安装包
